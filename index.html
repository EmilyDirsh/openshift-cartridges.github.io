What's a Cartridge?

A cartridge is a pluggable piece of a software stack that can be added to an application. Cartridges provide runtimes, databases, and middleware.

The OpenShift team builds and maintains a lot of the most popular runtimes and databases, but anyone who wants to use something less-well-known, brand new, really old, or just plain different, can build and share a community cartridge.

Use a Community Cartridge

Community cartridges have a URL where OpenShift can download the packaged cartridge. Once you know the URL, you can use it with the normal rhc commands.

Runtime cartridges

These are cartridges that provide an application runtime. Examples include Wildfly and Go. Applications can only use a single runtime cartridge, so these have to be added to a new application when it is created. 

rhc create app [URL]

Other cartridges

Non-runtime cartridges can be added anytime after the application is created. 

rhc cartridge add [URL]

Note: The web console also supports community cartridges, both creating an application from one and adding one to an existing application.
